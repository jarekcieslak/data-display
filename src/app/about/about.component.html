About project


Solution was developed using Angular 7 with some additional libraries. Data is served via REST API and exposed using json-server. Additional libraries used:

  • Chart.js (to generate charts)
  • Angular material (for consistent material design UI controls)
  • Twitter bootstrap (grid & css styling)

Installation

  1. npm install / yarn install in the main directory
  2. In additional terminal window run mock backend using npm run mock-server / yarn run mock-server.
    That should start local development server serving data via REST API. Server should run under http://localhost:3000.
  3. In order to start the development version of the application frontend run: npm run start / yarn run start
  4. In order to start the production build run: npm run start-prod / yarn run start-prod.
Remark: It's recommended to run production version, because the total app bundle is smaller and everything is working faster, but it's not possible to debug the app easily.

Assumptions & Remarks

  • Some further work would be necessary to remove js dependencies that are not needed on production build (Improve tree-shaking).
  • Twitter bootstrap could be replaced with a simpler framework like for example http://getskeleton.com/ or simple set of responsive scss mixins.
  • I haven't done any validation on min/max filters and from/to dates. So it's possible to select max that is smaller than min. Needs some further work
  • Due to limited amount of time for the assignment unit testing and e2e testing was not done.
